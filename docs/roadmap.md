# Stampport Roadmap

## Phase 1 — MVP

Goal:
Validate whether users enjoy collecting cafe, bakery, dessert, and restaurant stamps.

Features:
- Mock login
- Stamp creation
- Stamp acquired card
- EXP / level
- Badges / titles
- Weekly quests
- My Passport
- Kick points
- Share card UI

Success:
- User can create a stamp in under 30 seconds.
- Stamp result feels rewarding.
- Badge/title progress creates motivation for another visit.
- My Passport shows personal taste identity.

## Phase 2 — Trust and Community

Goal:
Make stamps more meaningful through real visit evidence and same-stamp holder content.

Features:
- Same-stamp holder reviews
- Place-specific stamp room
- Kick point voting
- Photo-based visit evidence
- Verification level UI

## Phase 3 — Real Verification

Goal:
Increase trust in stamps and badges.

Features:
- QR verification
- GPS verification
- Dynamic QR
- Receipt OCR
- Trust score
- Badge eligibility based on trust score

## Phase 4 — Recommendation

Goal:
Use stamp history to suggest where to go next.

Features:
- Taste clustering
- Similar-user based recommendation
- Area conquest rate
- Monthly taste report
- Nearby quest recommendation

## Phase 5 — Native App

Goal:
Move from web MVP to app-like field experience.

Features:
- Expo or React Native app
- Camera
- Location
- Push notifications
- QR scanner
- Native share
